作業手順書

作ったアプリを、渡せる形にして動かすまで

Visual Studio(ビジュアルスタジオ)というソフトで作ったアプリを、完成版として書き出し、フォルダごとコピーして、実際に起動するまでの手順です。全部で6ステップ、所要時間はだいたい10分です。

ゴール: 「Release(リリース)」という完成版を作る → できあがったフォルダをコピーする → 中の .exe ファイル(アプリ本体)をダブルクリックして起動する
1

Visual Studio でプロジェクトを開く

デスクトップまたはスタートメニューから紫色のアイコンの Visual Studio を起動します。開いたら「最近開いた項目」に目的のプロジェクト名があるので、それをクリックします。

起動画面の左側に、最近使ったプロジェクトの一覧が出ます。

2

「Debug」を「Release」に切り替える

画面の上のほうに 「Debug」と書かれた小さな選択ボックスがあります(緑色の▶ボタンの左あたり)。ここをクリックして 「Release」 に変えてください。

Debug は「作業中の仮の版」、Release は「人に渡す完成版」という意味です。

▾を押すと2つ出てくるので、下の「Release」を選びます。

3

完成版を作る(ビルドする)

画面いちばん上のメニューから ビルド をクリックし、出てきた一覧の ソリューションのビルド をクリックします。

数十秒待つと、画面の下のほうに 「ビルド: 1 正常終了」 と表示されます。これが「完成版ができました」の合図です。

下の「出力」欄に「1 正常終了」と出ればOKです。

「失敗」と出たら: そのまま進めず、作った本人か詳しい人に「ビルドが失敗した」と伝えてください。
4

できあがったフォルダを開く

画面右側の「ソリューション エクスプローラー」という欄で、プロジェクト名を右クリックし、エクスプローラーでフォルダーを開く を選びます。

フォルダが開いたら、そこから bin → Release と順にダブルクリックで入っていきます。「net8.0-windows」のような名前のフォルダがもう1つある場合は、その中まで入ってください。

bin(ビン)フォルダの中に Release(リリース)フォルダがあります。

5

Release フォルダを「まるごと」コピーする

Release フォルダそのものを右クリック → コピーして、渡したい場所(USBメモリ、デスクトップ、共有フォルダなど)に貼り付けます。

いちばん大事な注意: フォルダの中の .exe ファイルだけをコピーしてはいけません。まわりにあるファイル(.dll など)はアプリを動かす部品です。部品が欠けるとアプリは起動しません。必ずフォルダごとコピーしてください。

箱(フォルダ)ごと渡すイメージです。中身をバラして渡すと動きません。

6

アプリを起動する

コピーした Release フォルダを開き、中にある 「アプリ名.exe」をダブルクリックします。アプリの画面が開けば完了です!

ファイルがたくさんあって見つけにくいときは、「種類」の列が「アプリケーション」になっているものが本体です。

「.exe」が見えない設定のパソコンもあります。その場合は「種類: アプリケーション」を目印に。

こんな画面が出たら

「WindowsによってPCが保護されました」という青い画面が出た
知らないアプリを初めて起動するときの標準の確認画面です。「詳細情報」→「実行」の順にクリックすれば起動します。
「.NET のインストールが必要です」のような画面が出た
アプリを動かすための土台ソフトがそのパソコンに入っていません。画面に表示されるリンクをクリックしてインストール(無料)してから、もう一度 .exe をダブルクリックしてください。
ダブルクリックしても何も起きない
フォルダごとコピーできているか確認してください(.exe だけコピーすると起動しません)。それでもダメなら作った本人に連絡を。
